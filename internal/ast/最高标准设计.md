# kvlang AST 最高标准设计

---


---

## 零、定位与设计目标

**ast 是 kvlang 的单层 IR 类型体系。** 从语义推导结构，所有节点类型由语言特性决定，不存在 HIR/LIR 分层。

### 设计目标

| 目标 | 含义 |
|------|------|
| **单层 IR** | 同一 AST 贯穿 parser → lower → layoutcode → kvcpu，类型不变 |
| **类型强制不变量** | 编译期可检测的约束靠类型系统，不靠注释或运行时检查 |
| **零字符串语义** | 操作数类别、参数签名、条件表达式均为结构化字段 |
| **确定性变换** | lower pass 是纯函数：`*ast.File → *ast.File`，可测试可证明 |
| **单一遍历入口** | 所有 AST 遍历通过 `Walk`/`Inspect`，类型开关集中到一处 |
| **源码位置追踪** | 每个节点携带 `Pos{Line, Col}`，支持 IDE 集成和诊断定位 |

---
## 一、设计原则

1. **从语义推导结构**：节点类型由语言特性决定。
2. **类型强制不变量**：编译期可检测的约束靠类型，不靠注释。
3. **零字符串语义**：操作数类别、参数签名、条件表达式均为结构化字段，不存在字符串中。
4. **单层 IR**：kvlang 只有一层 IR（见根目录最高标准设计.md §二），不分 HIR/LIR。lower 包是对同一 AST 的变换 pass，输入输出类型相同（`*ast.File`）。
5. **单一遍历入口**：所有 AST 遍历通过 `Walk`，类型开关集中到一处。

---

## 二、操作数类型

kvlang 的操作数属于四个互不相交的地址空间：

| 类别 | 语法 | 含义 |
|------|------|------|
| `RelSlot` | `./x` | 当前帧相对路径（frame-local 变量） |
| `AbsSlot` | `/a/b` | kvspace 绝对路径 |
| `Ident` | `name` | 函数名 / 块标签（编译期符号） |
| `Literal` | `42`, `"hi"` | 编译期常量 |

```go
// operand.go

type OperandKind uint8

const (
    RelSlot  OperandKind = iota // ./x
    AbsSlot                     // /a/b
    Ident                       // name, label
    Literal                     // 42, "hello"
)

// Operand 是类型化的读操作数。
type Operand struct {
    Kind  OperandKind
    Value string
}

// WriteSlot 是写目标，只能是路径（RelSlot 或 AbsSlot）。
type WriteSlot struct {
    Kind  OperandKind // 必须 ∈ {RelSlot, AbsSlot}
    Value string
}
```

## 三、函数签名

// sig.go

// Param 表示有名称和可选类型标注的形参。
type Param struct {

## 四、核心节点

### 4.1 Instruction（叶节点，双层共用）

```go
// inst.go

// Instruction 是 kvlang 的统一指令形式：opcode(reads...) -> writes...
// 控制流本身也是指令：br(./cond, t, f)，call(funcName, args...)，return(./v)
// Instruction 同时实现 HIRStmt 和 LIRStmt。
type Instruction struct {
    Opcode string
    Reads  []Operand   // 输入：任意 OperandKind
    Writes []WriteSlot // 输出：只能是路径槽
    Pos    Pos
}
```

### 4.2 控制流节点

```go
// ast.go

type IfStmt struct {
    // Cond 是条件表达式，已解析为 Instruction。
    // Writes 为空——由 lower 填入临时槽（若条件为复合表达式）。
    // 简单槽引用（如 ./flag）表示为 Instruction{Reads: [RelSlot "./flag"]}。
    Cond *Instruction
    Then []Stmt
    Else []Stmt // 可为空
    Pos  Pos
}

type WhileStmt struct {
    Cond *Instruction
    Body []Stmt
    Pos  Pos
}

type ForStmt struct {
    Var  string  // 迭代变量名
    Iter Operand // 迭代路径（RelSlot 或 AbsSlot）
    Body []Stmt
    Pos  Pos
}

type BreakStmt    struct{ Pos Pos }
type ContinueStmt struct{ Pos Pos }
```

### 4.3 基本块

```go
// ast.go

// BlockStmt 是带标签的块，body 可含任意 Stmt。
// lower pass 将 IfStmt/WhileStmt 展开为 BlockStmt + br 指令后，
// body 只含 *Instruction，但类型不变。
type BlockStmt struct {
    Label string
    Body  []Stmt
    Pos   Pos
}
```

## 五、Stmt 接口与文件

### 5.1 Stmt 接口

```go
// Stmt 是所有语句节点的接口（单层 IR）。
// 实现：*Instruction, *BlockStmt,
//       *IfStmt, *WhileStmt, *ForStmt, *BreakStmt, *ContinueStmt
type Stmt interface {
    Node
    stmt()
    String() string
}
```

### 5.2 文件与函数

```go
// ast.go

type File struct {
    Funcs         []Func
    TopLevelCalls []*Instruction // def 块外的顶层调用
}

type Func struct {
    Sig  FuncSig
    Body []Stmt
}

func (fn Func) FullText() string // 重建完整源码文本
```

## 六、变换流水线

```
io.Reader
    │
    ▼ parser.ParseCode() → *ast.File
      · Scan → []Token → parser 结构体
      · parseFunc     → Func{Sig: FuncSig, Body: []Stmt}
      · parseIf       → IfStmt{Cond: *Instruction}（直接构造，不经字符串）
      · parseWhile    → WhileStmt{Cond: *Instruction}
    │
    ▼ lower.File(*ast.File) → *ast.File    （同类型变换 pass）
      · IfStmt/WhileStmt → BlockStmt + br 指令序列
      · evalCond(*Instruction)：若为简单槽引用直接用，否则填写 Writes
      · lower 不依赖 parser 包
    │
    ▼ layoutcode.WriteFunc(kv, pkg, ast.Func)
      · fn.Sig.String()       → kv.Set(/func/pkg/name, sig)
      · fn.Sig.Params/Returns → 直接绑定，无运行时重解析
      · BlockStmt.Body        → kv.Set(/func/pkg/name/<label>/[i,j], ...)
```

## 七、Walk / Visitor

```go
// walk.go

// Pos 携带源码位置，零值表示"无位置信息"。
type Pos struct{ Line, Col int }

// Node 是所有 AST 节点的基接口。
type Node interface {
    nodePos() Pos
    nodeEnd() Pos
}

// Visitor 在 Walk 遍历中访问单个节点。
// Visit 返回非 nil 时继续遍历子节点；返回 nil 时停止。
type Visitor interface {
    Visit(node Node) (w Visitor)
}

// Walk 先序深度优先遍历 node 的完整子树。
func Walk(v Visitor, node Node)

// Inspect 是函数式 Walk（类比 go/ast.Inspect）。
// f 返回 false 时停止对当前子树的遍历。
func Inspect(node Node, f func(Node) bool)
```

**展开规则**：

```
Instruction  → 叶（无子节点）
IfStmt       → Walk(Cond)，Walk(Then...)，Walk(Else...)
WhileStmt    → Walk(Cond)，Walk(Body...)
ForStmt      → Walk(Body...)
BlockStmt    → Walk(Body...)
BreakStmt    → 叶
ContinueStmt → 叶
```

## 八、不变量

### lower.File 调用前

1. `Func.Body` 可含任意 `Stmt`（含 IfStmt / WhileStmt / ForStmt）。
2. `IfStmt.Cond.Writes` 为 nil（由 lower 填入临时槽）。
3. `ForStmt` 原样保留，不展开。

### lower.File 调用后

1. `Func.Body` 只含 `*BlockStmt`，无 IfStmt / WhileStmt。
2. 每个 `BlockStmt.Body` 只含 `*Instruction`，不含嵌套块。
3. 每个 `BlockStmt` 的最后一条 `Instruction.Opcode` ∈ `{"br", "call", "return"}`。

### Instruction 写槽

1. `Writes` 中每个 `WriteSlot.Kind` ∈ `{RelSlot, AbsSlot}`。
2. 条件指令（`IfStmt.Cond`）在 lower 前 `Writes` 为 nil；lower 后若为复合表达式则 `Writes = [{RelSlot, "./_cond_N"}]`。

## 九、文件职责

```
operand.go   OperandKind / Operand / WriteSlot
sig.go       Param / FuncSig / FuncSig.String()
ast.go       Stmt 接口 / File / Func / Instruction / BlockStmt
             IfStmt / WhileStmt / ForStmt / BreakStmt / ContinueStmt
walk.go      Node / Pos / Visitor / Walk / Inspect
format.go    Format(w, *File)  — 实现 Visitor，走 Walk
dump.go      Dump(w, *File)    — 实现 Visitor，走 Walk
```

**单向依赖**：

```
format.go / dump.go
    → walk.go
    → ast.go → inst.go → operand.go
              → sig.go
```

## 十、完整类型汇总

```go
// operand.go
type OperandKind uint8
const ( RelSlot OperandKind = iota; AbsSlot; Ident; Literal )
type Operand   struct { Kind OperandKind; Value string }
type WriteSlot struct { Kind OperandKind; Value string }

// sig.go
type Param   struct { Name, Type string }
type FuncSig struct { Name string; Params, Returns []Param }
func (FuncSig) String() string
func (FuncSig) ParamNames()  []string
func (FuncSig) ReturnNames() []string

// walk.go
type Pos     struct { Line, Col int }
type Node    interface { nodePos() Pos; nodeEnd() Pos }
type Visitor interface { Visit(Node) Visitor }
func Walk(Visitor, Node)
func Inspect(Node, func(Node) bool)

// ast.go
type Stmt      interface { Node; stmt(); String() string }
type File      struct { Funcs []Func; TopLevelCalls []*Instruction }
type Func      struct { Sig FuncSig; Body []Stmt }
type Instruction  struct { Opcode string; Reads []Operand; Writes []WriteSlot; Pos Pos }
type BlockStmt    struct { Label string; Body []Stmt; Pos Pos }
type IfStmt       struct { Cond *Instruction; Then, Else []Stmt; Pos Pos }
type WhileStmt    struct { Cond *Instruction; Body []Stmt; Pos Pos }
type ForStmt      struct { Var string; Iter Operand; Body []Stmt; Pos Pos }
type BreakStmt    struct { Pos Pos }
type ContinueStmt struct { Pos Pos }
func (*Instruction) stmt(); func (*BlockStmt) stmt()
func (*IfStmt) stmt(); func (*WhileStmt) stmt(); func (*ForStmt) stmt()
func (*BreakStmt) stmt(); func (*ContinueStmt) stmt()
```

## 十一、禁止项

| 编号 | 禁止 | 理由 |
|------|------|------|
| A1 | AST 节点 import kvspace 或任何运行时包 | AST 是纯数据，不可耦合执行层 |
| A2 | 新增 HIR/LIR 类型分化 | 单层 IR 是 kvlang 的根基 |
| A3 | `Instruction.Reads`/`Writes` 使用裸 `string` 代替 `Operand`/`WriteSlot` | 结构化类型强制不变量 |
| A4 | 在 AST 节点上挂载运行时方法（如 `SetKV`） | 数据与行为分离；SetKV 已在 layoutcode |
| A5 | Walk 之外新增遍历入口 | 单一遍历入口确保所有 pass 一致性 |
| A6 | `FuncSig` 回退为字符串存储 | 结构化签名是编译期类型检查的基础 |
