# lower 最高标准设计

---

## 零、一句话定位

**lower 是 kvlang 的编译器降级 pass**：将 `*ast.File` 变换为另一个 `*ast.File`，
消除高级控制流节点（`IfStmt`/`WhileStmt`），展开嵌套表达式，
使输出可被 kvcpu **直接解释执行**。

纯函数：无副作用，无 I/O，无 kvspace 访问。输入输出类型相同，可串联、可测试、可证明。

---

## 一、输入 / 输出不变式

### 输入（来自 parser）

- `Func.Body` 可含任意 `Stmt`：`*Instruction`、`*IfStmt`、`*WhileStmt`、`*ForStmt`、`*BlockStmt`
- `*Instruction.Expr` 可为多层嵌套的表达式树（如 `a + b * c`）

### 输出（交给 layoutcode.WriteFunc）

| 不变式 | 说明 |
|--------|------|
| 无 `*IfStmt` / `*WhileStmt` | 全部展开为 `*BlockStmt` + `br` 指令 |
| 无嵌套表达式 | 每条 `*Instruction.Expr` 的所有 `Args` 均为叶节点（kvcpu 可直接执行） |
| `*ForStmt` 保留 | 待执行层迭代原语就绪后处理（P11） |
| `*BreakStmt` / `*ContinueStmt` 保留 | 同上 |
| 最后一条指令是控制转移 | 每个 `BlockStmt.Body` 以 `br` / `call` / `return` 结尾 |

## 二、两类变换

### 2.1 控制流降级（续体传递 CPS）

`IfStmt` → 4 个 `BlockStmt`，内层块提升为函数顶层兄弟节点（读写码结构）：

```
if(cond) { A } else { B }
// 后续语句 cont...

→ [preamble..., goto fn/_if_N]
  BlockStmt _if_N:    { evalCond; br(cond, fn/_then_N, fn/_else_N) }
  BlockStmt _then_N:  { lowerBody(A)[仅指令]; goto fn/_merge_N }
  BlockStmt _else_N:  { lowerBody(B)[仅指令]; goto fn/_merge_N }
  BlockStmt _merge_N: { cont[仅指令] }
  [A/B/cont 产生的内层块...] ← 提升到函数顶层（不嵌套）
```

`WhileStmt` → 3 个 `BlockStmt`，同样扁平化：

```
while(cond) { body }
// 后续语句 cont...

→ [preamble..., goto fn/_while_N]
  BlockStmt _while_N: { evalCond; br(cond, fn/_do_N, fn/_exit_N) }
  BlockStmt _do_N:    { lowerBody(body)[仅指令]; goto fn/_while_N }
  BlockStmt _exit_N:  { cont[仅指令] }
  [body/cont 产生的内层块...] ← 提升到函数顶层
```

块跳转用 `goto fn/label`（TCO 同帧跳转）；`br` 使用完全限定标签（含函数前缀），避免 TCO 后 `.func` 变更导致解析失败。

**读写码核心规则**：所有 `BlockStmt` 必须是函数体的直接子节点（兄弟关系），不允许嵌套。`splitInstsAndBlocks` 负责提升内层块。`appendReturn` 递归处理所有块的出口。

**用户书写块**：`lowerBody` 遇到用户显式写的 `BlockStmt`（如 `entry: {}`），自动在前面插入 `goto fn/entry` 确保 `[0,0]` 非空；用户写的裸 `return` 补全为 `return(./retvals...)`。

### 2.2 表达式展开

将嵌套表达式树递归拆平，每层引入一个临时槽，直到每条指令的操作数全为叶节点，kvcpu 可直接执行：

```
result <- a + b * c        （输入：两层嵌套）

→ _1 <- b * c              （临时槽，只赋值一次）
  result <- a + _1          （操作数全为叶节点）
```

临时槽名格式：`_N`（帧局部，`N` 单调递增，与块标签共用 `labelGen`）。

## 三、文件职责

单文件 `lower.go`。

| 函数 | 职责 |
|------|------|
| `File` / `Func` | 入口，遍历所有函数 |
| `lowerBody` | CPS 续体传递：遇 if/while 取后续语句为续体，递归展开 |
| `lowerIfWithCont` | if → 4块 + 内层块提升（读写码） |
| `lowerWhileWithCont` | while → 3块 + 内层块提升（读写码） |
| `splitInstsAndBlocks` | 将 `[]Stmt` 拆分为「指令」和「块」两组（用于提升） |
| `evalCond` | 条件求值：简单槽直接用；复合表达式展开并写入临时槽 |
| `flattenInstExpr` | 表达式树 → 可直接执行的指令序列（递归） |
| `appendReturn` | 递归为所有块补充隐式 return；处理裸 return 补全 |
| `isBareReturn` | 判断无参 return 指令（用于 appendReturn） |
| `preambleEndsWithTerminator` | 判断前缀末尾是否已有终止符（避免重复 goto） |
| `injectGoto` | 在块体末尾追加 goto（若末尾非终止符） |
| `isTerminator` | 判断 return/goto/br 终止符 |
| `brInst` / `gotoLabel` | 构造 br/goto AST 节点（标签均完全限定） |
| `labelGen` | 单调递增标签生成器，格式 `_prefix_N`；块和临时槽共用计数器 |

## 四、禁止项

| 编号 | 禁止 | 理由 |
|------|------|------|
| LW1 | lower import kvspace / keytree / layoutcode / kvcpu | 纯 AST 变换，不感知执行层 |
| LW2 | lower 修改 `FuncSig`（签名） | lower 只变换函数体，签名不变 |
| LW3 | 临时槽名复用或非单调递增 | 每个临时槽只赋值一次；复用导致 kvcpu 读到上一次执行的残留值 |
| LW4 | `lowerBody` 原地修改输入 `[]Stmt` | 纯函数契约；输入 AST 只读，输出是新节点 |
